# 二期：AI引导式互动求助系统

## Problem Frame

宿迁工会劳动争议咨询系统当前为 **L2 成熟度的单轮问答后端**（回答用户自由描述的问题，提供解答/建议）。二期需求是将系统升级为 **九步引导式全流程智能咨询应用**，覆盖从进门到文书生成的全链路。核心变化：从"自由对话获取建议"转向"结构化采集+智能化辅助"。

**用户故事：** 一位宿迁工厂工人，被欠薪3个月、不签劳动合同、被口头辞退。他打开系统，通过点选和填空而非打字，在20分钟内完成：案由定性 → 证据准备清单 → 仲裁申请书下载 → 维权路线图。

## Requirements

### R1. 模式选择入口
- 首页两个入口：**律师视频**（工作日直连值班律师）、**AI问答**
- AI问答支持语音输入、文字输入、资料上传
- 两个入口可随时切换

### R2. 问题初判（三路径）
- **路径A - 找律师视频**：跳转视频通话，仅工作日服务时段可接通
- **路径B - 自述案情**：文本框自由描述 → 系统基于300+案由追问厘清事实
- **路径C - 交互问答求助**：初始展示6个高频诉求图标（欠薪/开除/工伤/调岗/社保/其他）→ 12个通用共性问题全点选回答

### R3. 通用问题规范化采集
- 所有问题采用点选/下拉/日历选择，无手动输入
- 入职时间：日历选择 + "记不清"选项
- 工作岗位：下拉含新业态（外卖员/网约车司机）
- 金额计算：系统内置引导式计算器（加班费、经济补偿金等）

### R4. 特殊问题智能追问
- 系统根据路径B或C结果，自动生成有逻辑顺序的特殊问题清单
- 多重案由按优先级排序展示
- 规则同R3（点选/下拉/日历/计算器）
- **动态停止**：一旦厘清核心事实即停止提问

### R5. 案件初步定性
- 自动生成完整规范案件事实描述，会员可查看确认/返回修改
- 案由初步判定（单一或组合），标注案由名称及三级案由编码
- 提供法律依据索引按钮（跳转法规查询）
- **自动出具权益清单**：仲裁请求 + 赔偿/补偿金额及计算依据（通俗无术语）

### R6. 证据攻略系统
- **证据知识学习**：按案由匹配专属证据清单，分"必备准备"和"加强证据"，含通俗解释
- **证据梳理**：会员手动标记已有/可补充/无法取得的证据，系统记录状态
- **证据收集**：颜色标注状态（A=已有/B=可补充/C=无法取得），点击展开针对性解决方案（含话术模板、法律文书草稿如《要求出具劳动关系证明的函》）
- **证据上传**：拍照/相册上传，按类型分类存储，可查看/删除/替换
- **证据审核反馈**：智能审核 + 针对性评价，顶部实时展示收集完整度（充分/不完整/缺乏/严重缺乏四级）

### R7. 风险提示
- 结合案件事实清晰度、证据完整性、对方可能抗辩进行全面分析
- 明确指出具体风险点（时效过期/证据链断裂/诉求计算错误等）
- 高风险点醒目标注 + 基础规避建议

### R8. 文书生成
- **自动生成**：点击"生成文书"→ 前序数据自动填入模板，支持《仲裁申请书》《调解申请书》
- **智能校验**：对事实不清晰项高亮标注 + 弹出补充问题窗口
- **会员主动提问修改**：基于知识库对请求/理由部分修改或优化建议
- 暂不实现：手动编辑修改、Word/PDF导出

### R9. 行动路线图
- 流程图展示"协商→调解→仲裁"维权核心步骤
- 点击每步卡片：操作指引 + 准备材料 + 宿迁本地办理点地址电话 + 最佳操作时间
- 适宜调解案件：提示"优先尝试工会调解，电话：XXX"
- 所有地址支持导航跳转

### R10. 求助复核
- **AI开放复核**：打包案件信息+证据+文书，窗口提供预设提示词模板 + 自由输入
- **一键求助律师**：任意环节点击"求助工会律师"，打包所有已收集数据发送至市总工会值班律师端，同时发起咨询申请

## Success Criteria

- 会员在无人工干预情况下，通过点选填空在30分钟内完成从进门到生成仲裁申请书的全流程
- 案由定性准确率：通过二期案由分类测试集（300+案由）
- 权益清单覆盖率：基于案由应出权益与实际输出权益对比 ≥ 95%
- 证据审核反馈准确率：人工抽检准确率 ≥ 90%
- 文档生成完整性：模板填充后需补充的信息高亮率 ≤ 5项/文书

## Scope Boundaries

- **不实现**：律师视频通话功能（仅做入口跳转）
- **暂不实现**：文书手动编辑修改（仅R8部分）、Word/PDF导出
- **暂不实现**：非工作时段自助维权路径指引
- **不实现**：宿迁以外地区办理点信息

## Key Decisions

- **LangGraph升级为九步主流程**：每一跳都是LangGraph中的独立节点/状态机，而非新增API层。理由：用户明确选定此架构方向。
- **按九步顺序逐阶段构建**：分阶段实施（R1-R10逐步），降低风险。
- **数据模型优先建设**：案由分类体系（300+案由 × 共性12题 × 个性问题库）是整个流程的基础数据，需最先定义。
- **当前Agent逐步演进**：现有answerer/detailer/advisor/summarizer将逐步演进为九步中的特定节点，而非推倒重来。

## Dependencies / Assumptions

- LLM API可用（OpenAI兼容，配置不变）
- 案由分类以《民事案件案由规定》为基础，补充自定义新业态案由；先从6大高频分类（欠薪/开除/工伤/调岗/社保/其他）开始，其余300+案由后续迭代
- 12个通用共性问题已确认（见下方）
- 证据上传采用本地文件存储，按案件ID组织目录
- 文书模板采用国家标准格式（劳动仲裁申请书通用格式），无需特殊本地化
- 办理点信息暂用省级/国家层面通用信息（宿迁本地数据待后续填充）

## 12个通用共性问题（已确认）

| # | 问题 | 输入方式 |
|---|---|---|
| Q1 | 您目前的就业状态？ | 点选：在职/离职/待岗 |
| Q2 | 离职时间？ | 日历选择 + "记不清" |
| Q3 | 是否签订书面劳动合同？ | 点选：是/否/不确定 |
| Q4 | 每月工资大概多少？ | 数字输入 |
| Q5 | 工资发放方式？ | 点选：银行转账/现金/微信/支付宝 |
| Q6 | 公司是否为您缴纳社保？ | 点选：是/否/部分 |
| Q7 | 工作岗位是？ | 下拉：工厂工人/外卖员/网约车司机/办公室/其他 |
| Q8 | 入职时间？ | 日历选择 + "记不清" |
| Q9 | 每周工作时间？ | 点选：标准工时/加班/不定时 |
| Q10 | 涉及哪些诉求？ | 多选：欠薪/加班费/赔偿金/经济补偿/社保/其他 |
| Q11 | 涉及金额大概多少？ | 数字输入，可选"不确定" |
| Q12 | 您希望的维权结果？ | 点选：继续上班/拿钱走人/补偿金 |
