
# 智能劳动维权咨询系统：技术实现路径


## 一、案由定性：基于语义解析与向量化建模的推理机制

系统构建了一套**“感知-认知-校验”**的闭环架构，将用户的非结构化陈述转化为具备法律意义的结构化数据。

* **语义解构：** 采用大语言模型作为认知引擎，通过**Few-shot Learning（少样本学习）**技术，将用户的模糊表达映射为标准的领域本体，提取关键要素和信息。
* **向量空间检索：** 采用高维向量建模技术，将案情向量化，并在向量数据库中与预定义的劳动争议案由体系进行**空间距离计算**。该方法能够排除自然语言的歧义，通过数学逻辑精准锁定案由。
* **针对性追问：** 系统并非简单的“黑盒”映射，而是基于业务逻辑的判定规则集，LLM根据信息收集进度和可能案由候选，针对潜在模糊点自动触发追问逻辑，确保追问过程具备案由业务逻辑。

## 二、路径选择：多维评分矩阵与动态决策算法

针对维权路径（协商、调解、仲裁）的选择，系统引入了**多维评分决策模型**，将法律实务经验转化为可量化的算法逻辑。

* **路径适配评分矩阵：** 通过引入加权因子，评估案件的“胜算潜能”与“处理成本”。算法综合考量证据链完整度、案件时效紧迫度、赔偿金额的法律基准等因子，通过**加权求和**算法输出量化的适配分值。
* **动态状态机：** 系统维持一个状态机模型，实时监控维权进展。若用户在某路径受阻（如协商失败），系统将自动触发状态更新，基于更新后的数据重新计算各路径的权重，实现决策路径的闭环动态重构。

## 三、信息呈现：基于多智能体协作与RAG的智能交互

为优化用户认知负荷，系统采用**分层展示架构**，使用**多智能体**并通过**RAG**技术确保内容的专业度与合规性。

* **多智能体协调（Multi-Agent Handoffs）：** 系统内部署不同的Agent负责结论分析、法规检索与行动建议，通过智能调度与自动摘要机制，将繁杂的法律条文凝练为核心结论。
* **证据溯源机制：** 结论层（直观呈现）依托确定性计算；依据层（法律溯源）通过RAG技术从法律数据库中精准抓取原文，消除了模型虚构（Hallucination）风险；行动层（动态指引）则根据任务进度实时渲染。

## 四、文书生成：结构化模板与逻辑计算的深度融合

针对不同类型的文书，系统采取了差异化的**生成式逻辑策略**：

* **语义转译引擎：** 在生成仲裁申请书等文书时，系统通过预设的模板引擎（Template Engine），将提取的自然语言事实自动转译为严谨的法言法语，确保文书的专业规范性。
* **确定性算子引擎：** 针对赔偿金额、未休年假工资等涉及财务计算的文书，系统抛弃生成式方法，改用内置的硬编码计算逻辑，确保计算精度为100%，杜绝任何算术误差。
* **业务规则集：** 证据清单与文书完整性校验不再依赖模型“直觉”，而是由内嵌的业务规则集进行断言判断，确保每一份输出的文书均符合法定格式要求。
