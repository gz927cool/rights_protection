# AI问答流程入口路径理解澄清

## 问题描述

在实现"问题初判"步骤时，最初理解认为智能体需要询问用户选择三种处理路径：
- A) 转律师视频
- B) 自由描述案情
- C) 交互式问答

这一理解导致 `step2_initial` 的 prompt 中错误地包含了路由选择逻辑。

## 根本原因

对需求文档的误解：

1. **首页**已有顶层入口选择（律师视频 / AI问答），用户通过点击按钮做出选择
2. **问题初判页面**展示三个按钮，但"找律师视频"是跳转动作，不属于AI流程
3. AI问答内只有**两种**形式：自述案情 和 交互问答——这是前端UI差异，智能体不应感知

## 正确理解

### 架构分层

```
┌─────────────────────────────────────────────────────────┐
│  首页                                                      │
│  ├── [律师视频] ──────────→ 视频通话模块                      │
│  └── [AI问答]  ──────────→ AI智能体流程                     │
│                            │                               │
│                 ┌───────────┴───────────┐                 │
│                 ↓                       ↓                 │
│          【自述案情】              【交互问答】               │
│          聊天框自由输入             UI填表/选择题            │
│                 │                       │                 │
│                 └───────────┬───────────┘                 │
│                             ↓                               │
│                    统一的 Agent 智能体                     │
│                    （只处理内容，不感知UI形式）              │
└─────────────────────────────────────────────────────────┘
```

### 职责边界

| 层次 | 职责 | 示例 |
|------|------|------|
| **前端/展现层** | 入口路由、按钮点击分发、UI渲染 | React组件 |
| **智能体/业务层** | 内容理解、案由提取、权益生成、文书撰写 | LangGraph |

**关键约束**：
- 智能体**不询问**用户选择哪个入口路径（A/B/C）
- 前端通过按钮点击做路由，智能体只接收已路由的内容
- "自述案情"和"交互问答"是前端UI差异，智能体**不感知**输入形式
- 智能体唯一职责：处理用户输入的内容，完成业务逻辑

## 解决方案

### 代码修改

**文件**: `langgraph_model/consultation_graph.py`

移除了 `step2_initial` prompt 中错误的 A/B/C 路由选择逻辑：

```python
# 已删除的错误逻辑：
# 如果用户输入不包含路由关键词：
# → 输出"请选择处理方式：A)转律师视频 B)自由描述案情 C)交互式问答"

# 正确的逻辑：
# - case_category_extracted = true 时，直接进入下一步
# - 不再询问 A/B/C 路径选择
```

### 前端-智能体接口约定

前端将所有输入统一转化为结构化文本传给智能体：

```typescript
interface UserInput {
  content: string;           // 用户输入的文本
  inputType: "text" | "voice" | "form";
  formData?: Record<string, string>;
  currentStep: string;       // 当前步骤
}
```

智能体通过 `currentStep` 感知进度，通过 `content` 理解内容，**不感知 inputType**。

## 预防策略

1. **明确职责边界**：入口选择是前端职责，智能体只处理内容
2. **架构设计时区分**：将"前端路由决策"和"后端业务处理"严格分开
3. **代码审查检查项**：智能体 prompt 中不应包含路由选择逻辑
4. **文档固化**：将智能体架构边界约定写入 CLAUDE.md（已执行）
