# 需求理解与澄清

## 一、核心入口澄清

### 1.1 模式选择层（首页）

用户进入系统后，首先面临**两个**顶层入口选择：

| 入口 | 说明 | 后续流程 |
|------|------|----------|
| **律师视频** | 直连值班律师，仅工作日服务时段开放 | 跳转视频通话模块 |
| **AI问答** | 对话框支持语音、文字输入及资料上传 | 进入AI智能体流程 |

**关键点**：此层为**前端页面按钮**，用户点击即选择，智能体不参与决策。

### 1.2 问题初判页面的三个入口

问题初判页面展示三个选项按钮：

| 形式 | 用户体验 | 说明 |
|------|----------|------|
| **找律师视频** | 跳转视频模块 | 从AI流程切换到视频咨询（非AI流程的一部分） |
| **自述案情** | 聊天框自由文字输入 | 类似对话，输入框直接打字 |
| **交互式问答** | UI填表/选择题 | 12个通用问题，逐题点选 |

**重要澄清**：

1. "找律师视频"是**跳转动作**，点击后离开AI流程，进入视频模块
2. AI问答流程内实际只有**两种形式**：自述案情 和 交互问答
3. 这两种形式是**前端交互方式**的差异，不是智能体的两种处理路径
4. 智能体本身**不询问**用户选择哪种形式 — 前端根据用户点击的按钮来决定渲染哪种UI

## 二、智能体职责边界

### 2.1 "找律师视频"的定位

"找律师视频"按钮位于问题初判页面，但它**不属于AI问答流程**：
- 用户点击 → 离开AI流程 → 跳转视频咨询模块
- 智能体无需感知此跳转

### 2.2 智能体只做信息处理

```
用户输入（文字/语音/表单数据）
        │
        ▼
┌─────────────────────────┐
│   统一的 Agent 智能体   │  ← 只感知"用户说了什么"
└─────────────────────────┘
        │
        ├── 提取案由
        ├── 理解案情
        ├── 生成权益清单
        ├── 出具风险提示
        └── 生成文书
```

智能体**不关心**用户是通过打字还是点选方式输入的，只关心输入的内容。

### 2.3 前端负责交互路由

| 前端职责 | 智能体职责 |
|----------|------------|
| 渲染两种不同的交互UI（聊天框/填表） | 接收结构化或非结构化文本 |
| 根据用户点击分发到不同组件 | 理解并处理内容 |
| 展示权益清单、证据状态等结果 | 生成结构化数据供前端展示 |

## 三、当前代码与需求的偏差

### 3.1 step2_initial 的问题

当前实现中，`step2_initial` 的 prompt 包含了：

```
如果用户输入不包含路由关键词：
→ 输出"请选择处理方式：A)转律师视频 B)自由描述案情 C)交互式问答"
```

**这是错误的**，原因：

1. 用户进入AI问答后，**已经**通过前端点击选择了某种形式
2. 智能体不应再询问A/B/C
3. 应该直接进入对应的信息收集流程

### 3.2 正确理解

```
首页
  └── 点击 [AI问答]
            │
            ▼
    前端渲染：
    - 自述案情模式 → 聊天框
    - 交互问答模式 → 12问填表UI
            │
            ▼
    用户输入内容 → 智能体处理
```

智能体收到的是**已经过前端路由的输入内容**，不需要再判断用户想要走哪条路径。

## 四、架构建议

### 4.1 分层清晰

| 层次 | 职责 | 示例 |
|------|------|------|
| **展现层** | 前端UI渲染、用户交互、路由分发 | React组件 |
| **业务层** | 智能体逻辑、案由提取、权益生成 | LangGraph |
| **数据层** | 状态管理、持久化、存储 | FastAPI后端 |

### 4.2 前端与智能体接口

前端将用户输入（无论是自由文字还是表单数据）**统一转化为文本**传给智能体：

```typescript
// 前端统一格式
interface UserInput {
  content: string;           // 用户输入的文本
  inputType: "text" | "voice" | "form";
  formData?: Record<string, string>;  // 如果是表单，扁平化为key-value
  currentStep: string;        // 当前步骤
}
```

智能体无需感知输入来自哪种UI，只处理 `content`。

## 五、总结

1. **首页A/B选择**是用户主动点击，智能体不参与
2. **"找律师视频"**是跳转动作，不属于AI问答流程
3. **AI问答内两种形式**（自述案情/交互问答）是前端UI差异，智能体不感知
4. **智能体唯一职责**：处理用户输入的内容，完成案情理解、案由判定、权益清单生成等业务逻辑
5. **前端与智能体解耦**：前端负责交互，智能体负责思考
